Java反射

发表于 2020-03-07

反射

Class

首先反射的基石 Class 类
Java 中每个类都属于同一个事务，而描述这一事务的Java类就是Class，当编译一个新类的时候，会产生一个.class文件，而这个文件里存的就是Class对象。
Class 包含了描述类的信息，比如

  • 类的名字
  • 类的访问属性
  • 类的字段
  • 类的成员方法
  • ……

而反射就是在运行阶段，动态的通过Class 类去访问和调用类的属性和方法。

  • 静态编译：在编译的时候绑定对象，确定类型
  • 动态编译：在运行的时候绑定对象，确定类型
    动态编译可以最大程度的支持多态，而多态的最大意义在于解耦，因此反射的重要性也就体现出来了，主要是用于降低程序的耦合度，提高代码的灵活性。
    但是动态编译因为是在运行是才绑定对象，这也导致了性能比直接调用对象慢很多。

一般获取Class对象有三种方法

  • Class class = student.class 通过类名直接获取
  • Class class = new student().getClass() Object类中的getClass()方法获取
  • Class class = Class.forName("student") 通过静态方法去获取该字符串对应的类的字节码。(最主要的一种方式)
    通过这三种方式获得的字节码都是相同的(通过$==$ 表示相同)，一个Class对象实际上表示的是一种类型，也有可能是基本类型比如int.class。

一般来说有三大类是比较重要的

  • Field 字段，可以通过get(),set() 方法读取和修改对象关联的字段
  • Method 方法，可以通过invoke() 方法调用对象的方法
  • Constructor 通过new instance() 方法创建新的对象

反射的应用场景

  • 通过反射运行配置文件的内容

  • JDBC 的数据库的连接

在JDBC 数据连接的时候，需要先用Class.forName() 去加载驱动，然后通过DriverManager类进行数据库连接，然后connect接口进行接收drivermanager.getConnection()。

  • Spring框架的使用

    反射是Java框架的基石
    在Sprng中比较典型的就是xml的配置模式
    Spring通过XML配置模式装载Bean的过程
  1. 将配置文件或者是XML文件装载到内存中
  2. 读取并解析配置文件，获得所需对象的字节码字符串以及相关属性信息
  3. 通过反射机制，获取到所需对象的Class实例
  4. 动态配置实例的属性

这样做如果需要修改的话，直接修改配置文件即可，不用每次在代码中new。
