Java集合框架

发表于 2020-02-25

Java 集合框架


三大集合接口

对于集合框架来说，上层接口有两个Collection和Map
Collection下层又有两个子接口分别是List和Set
这样就构成了Java的三大集合 Map,List,Set
然后往下细分就有很多了。

Map

首先是Map，Map中主要存储的是键值对信息。

HashMap

首先要了解HashMap的结构，在jdk1.8以前HashMap的主要通过数组+链表的结构，而之后是以数组+链表+红黑树的结构。
当链表长度大于8时，链表会转换成红黑树，而长度降为6时，会退化成链表
(这里有一个很奇葩的问题是为什么要选择8作为链表转红黑树：因为TreeNode的空间是普通Node的两倍，所以尽量不希望转换成红黑树，而根据理想情况下随机hashCode算法下所有bin中节点的分布频率会遵循泊松分布，当链表长度为8的时候，可能性已经很小很小了，基本上是不可能事件，所以选择了8)
(对于为什么选择红黑树而不是AVL树：因为红黑树具有良好的稳定性和完整的功能，便于维护，在插入和删除的时候效率高于AVL树)

当然对于hashMap来说，是线程不安全的，而如何体现出线程不安全的呢。有两种例子，一是在扩容的时候，hashMap在JDK1.8之前要根据新长度进行重新hash分配。但是在JDK1.8之后就简化了这一步骤，就是因为长度是原长度的两倍，对(长度-1)取模之后，这个位置只可能在原位置不变或者是出现在2次幂*原位置的地方。而依据则是转换成二进制之后，新增那一位的取值，如果为0则在原位置不变，反之出现在原索引+原长度的位置上。在jdk1.7的时候，在扩容的时候会将链表置换，导致死循环，但是到了jdk1.8之后优化了，也不会了。但是在进行一些操作的时候，还是会出现数据丢失的情况。二就是fast-fail机制 即快速失败机制 是基于底层集合做拷贝的，当多线程对Collection 进行操作的时候，若一个线程要用迭代器对内容遍历集合的时候，该集合的内容被其他线程进行修改，会抛出 ConcurrentModificationException。

解决hash冲突的方法
1.链式法，就是在hash数组后面将hash值相同的链在一起（hashmap所用的）
2.再散列法 其中有三种形式，分别是线性探测再散列（即在数组后面+1+2+3…判断是否有空位） 二次探测再散列法 （去探测当前位置+1 -1 +2 -2 +4 -4 +$k^2$ - $k^2$是否有空位) 还有伪随机再散列法（建立一个伪随机生成器）
3.再Hash法 构造多个hash函数，在冲突的时候使用，直到不冲突，增大计算时间

HashTable

线程安全：因为对put，get等操作都加了sync函数，但是在操作的时候，将整个集合都锁住了，导致效率不高
但是在进行复合操作的时候（比如迭代，导航，一个put，一个remove）还是存在数据丢失等情况。

ConCurrentHashMap

集合了HashTable和HashMap 的 优势的一个 线程安全容器

但是ConCurrentHashMap 不能插入一个Null值
而HashMap 能插入一个唯一的Null值.
使用分段锁，也不会像HashTable一样锁住整个表结构。
对于分段锁
可以理解将整个Hash数组分为16（初始值,并发度为16)个桶，然后要访问桶内的数据，必须获取这个桶的锁。
在JDK1.7版本 结构还是一个HashEntry[] 一个Segment[] Segment是可重入锁ReenTrantLock结构时类似数组+链表的形式。一个Segment 里面包含一个HashEntry数组，如果要访问里面的元素就必须获取当前Segment的锁
在JDK1.8之后改成了sync和CAS进行Node锁，主要是避免ReenTrantLock 继承AQS机制，同步，消耗内存，在1.8之后只有头结点需要同步。

TreeMap

底层结构:红黑树
特点:可以通过键值进行排序的Map

LinkedHashMap

元素继承于HashMap中的 Node 但是 新增了两个属性，即 before和after 双向链表，确定插入顺序和访问顺序

可以使用LinkedHashMap 实现LRU 页面置换算法（最近最久未访问）
操作为 首先将LinkedHashMap 的accessOrder 置为true （表示使用访问顺序，false默认为插入顺序）
然后重写removeEldestEntry函数即可

List

Collections和Collection的区别

Collections 是 集合的最上层集合接口
而Collection 是 集合的工具类 其中包括sort，binraySearch,reverse,min(max) 等方法

ArrayList

基于动态数组的集合，随机存取效率高。
默认容量为10，每次扩容为当前容量*3/2+1.
需要在创建的时候预留一些空间扩容。

Array和Arrays 的区别

Array是一个基本的存储结构
Arrays 是一个静态类，其中保存了一些数组方法，包括sort，binarySearch，equals(判断两个集合是否类型一致，以及内容是否一致，与== 和 .equals 是判断引用是否一样)。

LinkedList

基于双向链表的集合，插入删除效率高
空间方面对于每个元素都需要额外的空间去保存信息以及指针信息。

CopyOnWriteArraylist

是基于COW（Copy-On-Write) 思想的一个并发容器，首先是CopyOnWrite是当有线程想往容器中添加数据的时候，会先将当前容器进行Copy，复制出一个新的容器，然后再往新的容器里添加元素，在添加元素完成后，在将旧容器的引用改成新容器的引用，这样做的好处是可以实现并发的读，而不需要加锁，因为读写分离。
但是在写的过程中还是需要加锁的，不然会复制出n个副本。

看源码可以看出来，在写操作的时候用来sync锁住写操作。
当然对于CopyOnWriteArrayList 来说有两个要注意的问题
1.内存占用的问题，因为写复制机制，内存中里存在两个对象的内存，而如果这些对象占用的内存比较大的时候，会造成频繁的YONG GC和FULL GC。
2.数据一致性问题，因为CopyOnWriteArrayList只保证数据的最终一致性，而没有保证数据的实时一致性。

Set

LinkedHashSet

底层还是使用的LinkedHashMap实现

HashSet

使用的是hash表实现
底层还是使用的HashMap，但是只使用了Key值 value = PRSENT ，当重复数据进来的时候，只是覆盖value，保证了数据不重复。
保证数据的唯一性：先判断hashcode，当hashcode相同的时候再判断equals

TreeSet

使用了红黑树实现，保证数据唯一性，通过比较器。
